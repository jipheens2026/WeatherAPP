## 📦 Installation & Setup

### Prerequisites
- Node.js 18+ and npm/yarn
- Weather-AI API key (get one at [weather-ai.co](https://weather-ai.co))
- Weather-AI developer docs: [https://weather-ai.co/docs](https://weather-ai.co/docs)

### Step 1: Clone the Repository
```bash
git clone https://github.com/yourusername/weather-ai-dashboard.git
cd weather-ai-dashboard
```

### Step 2: Install Dependencies
```bash
npm install
```

### Step 3: Configure Environment Variables
Create a `.env` file in the project root:
```bash
cp .env.example .env
```

Edit `.env` and add your Weather-AI API key:
```
VITE_WEATHER_AI_API_KEY=wai_your_actual_api_key_here
```

⚠️ **Important**: Never commit your `.env` file to version control. It's already listed in `.gitignore`.

### Step 4: Run the Development Server
```bash
npm run dev
```

The application will be available at `http://localhost:5173`

### Step 5: Build for Production
```bash
npm run build
```

The optimized production build will be in the `dist/` folder.

## 🌐 Deployment

### Deploy to Netlify

1. **Install Netlify CLI**:
   ```bash
   npm install -g netlify-cli
   ```

2. **Build and Deploy**:
   ```bash
   npm run build
   netlify deploy --prod --dir=dist
   ```

3. **Configure Environment Variables**:
   - Go to Netlify Dashboard → Site Settings → Environment Variables
   - Add `VITE_WEATHER_AI_API_KEY` with your API key
   - If the site was already deployed, trigger a new deploy after saving this variable

> Note: In production, the app uses a Netlify serverless proxy to forward requests to Weather-AI and avoid browser network/CORS failures.

### Deploy to Vercel

1. **Install Vercel CLI**:
   ```bash
   npm install -g vercel
   ```

2. **Deploy**:
   ```bash
   vercel --prod
   ```

3. **Configure Environment Variables**:
   - Go to Vercel Dashboard → Project Settings → Environment Variables
   - Add `VITE_WEATHER_AI_API_KEY` with your API key

### Deploy via GitHub (Recommended)

1. Push your code to GitHub
2. Connect your repository to Netlify or Vercel
3. Configure build settings:
   - **Build Command**: `npm run build`
   - **Publish Directory**: `dist`
4. Add environment variables in the platform dashboard
5. Deploy automatically on every push to main

## 🎯 Usage Guide

### Weather Search

1. **Quick Search**: Click one of the pre-configured city buttons (Nairobi, London, etc.)
2. **Custom Location**: Enter latitude and longitude in the input fields and click "Get Weather"
3. **View Results**:
   - Current conditions with temperature, feels-like, humidity, wind speed
   - AI-generated weather summary from Gemini
   - 7-day forecast cards
   - Interactive temperature trend chart
   - Precipitation forecast bar chart

### Tree Analysis

1. **Upload Image**: 
   - Click the upload area or drag & drop an aerial/drone image
   - Supported formats: JPEG, PNG, WEBP (max 20MB)
2. **Add Context** (Optional but recommended):
   - Farmer ID (e.g., F-001)
   - County/Region (e.g., Bomet)
   - Land size in acres (e.g., 2.5)
   - Farm location/name
   - Additional notes about the plantation
3. **Analyze**: Click "Analyze Trees" and wait for processing
4. **Review Results**:
   - Total tree count and density metrics
   - Canopy coverage percentage
   - Health distribution (healthy/needs care/needs replacement)
   - AI observations about canopy conditions
   - Actionable recommendations
   - Side-by-side original and annotated images
